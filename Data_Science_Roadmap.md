# 🗺️ Data Science Learning Roadmap
### For: Some coding + some math background → Job-ready

---

## Stage 1 — Python for Data Science
**⏱ Estimated time: 3–4 weeks**

Since you already know some Python, this stage is about sharpening your skills for data work specifically. Focus on NumPy, Pandas, and Matplotlib — these are the daily tools of any data scientist. You should be comfortable loading datasets, cleaning messy data, and creating basic visualizations before moving on.

**Free Resources:** Kaggle's free Python & Pandas courses (kaggle.com/learn), W3Schools Python, freeCodeCamp YouTube

**Project:** Load a real CSV dataset (e.g. Titanic from Kaggle), clean it, and visualize key insights.

---

## Stage 2 — Statistics & Probability for Data Science
**⏱ Estimated time: 3–4 weeks**

Since you have some college stats, this will be a refresher with a practical lens. Focus on descriptive stats, distributions, hypothesis testing, correlation, and probability basics. These concepts come up constantly in interviews and real work.

**Free Resources:** StatQuest with Josh Starmer (YouTube) — absolutely essential, Khan Academy Statistics

**Project:** Perform an A/B test analysis on a sample dataset and write up your findings.

---

## Stage 3 — Data Cleaning & Exploratory Data Analysis (EDA)
**⏱ Estimated time: 2–3 weeks**

This is where most real data science time is spent. Learn how to handle missing values, outliers, feature engineering, and how to ask good questions from data. EDA is often what separates junior from mid-level data scientists.

**Free Resources:** Kaggle notebooks (study others' EDA notebooks), Towards Data Science (Medium)

**Project:** Pick any Kaggle dataset and write a full EDA report — visualizations, observations, and a summary of insights.

---

## Stage 4 — SQL
**⏱ Estimated time: 2–3 weeks**

SQL is non-negotiable for any data science job. You'll use it to pull and manipulate data from databases. Focus on SELECT, JOINs, GROUP BY, subqueries, and window functions.

**Free Resources:** SQLZoo, Mode Analytics SQL Tutorial, LeetCode (SQL section for interview prep)

**Project:** Use a free database like the Northwind or Chinook dataset and answer 10 business questions using SQL queries.

---

## Stage 5 — Machine Learning Fundamentals
**⏱ Estimated time: 6–8 weeks**

This is the core of data science. Learn supervised learning (regression, classification), unsupervised learning (clustering), model evaluation, overfitting, cross-validation, and feature selection. Use scikit-learn as your main library.

**Free Resources:** Andrew Ng's ML Course on Coursera (audit for free), Kaggle ML courses, Hands-On ML book (free online)

**Project:** Build a house price prediction model (regression) and a customer churn classifier — two classic job interview projects.

---

## Stage 6 — Data Visualization & Storytelling
**⏱ Estimated time: 2 weeks**

Knowing the numbers isn't enough — you need to communicate them. Learn Matplotlib, Seaborn, and optionally Plotly for interactive charts. Focus on making charts that tell a clear story to a non-technical audience.

**Free Resources:** Kaggle Data Visualization course, Storytelling with Data blog (storytellingwithdata.com)

**Project:** Take one of your earlier projects and build a clean, presentation-ready visual report.

---

## Stage 7 — Build Your Portfolio & Prep for Jobs
**⏱ Estimated time: 4–6 weeks (ongoing)**

This is where everything comes together. Put your projects on GitHub, write about them on LinkedIn or Medium, and start applying. Practice interview questions — both technical (stats, ML concepts, SQL) and behavioral.

**Free Resources:** GitHub (free portfolio hosting), Kaggle competitions for experience, interviewing.io for mock interviews, Glassdoor for common DS interview questions

**Must-do:** Have at least 3 solid end-to-end projects on GitHub before applying.

---

## ⏱ Total Estimated Timeline
**~5 to 6 months** studying consistently (1–2 hours/day)

---

## 💡 Pro Tips for Getting Hired
- Kaggle competitions look great on a resume even without winning
- Write a short blog post for each project — it shows communication skills
- Apply to data analyst roles first — it's a great entry point into the field
- Tailor your resume to each job using keywords from the job description
